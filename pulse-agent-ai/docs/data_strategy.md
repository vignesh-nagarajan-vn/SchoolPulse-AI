# Data Strategy

## Data Used Now

The MVP uses synthetic data with realistic school constraints:

- Hourly energy readings by zone.
- Event schedules and outcomes.
- Water leak alerts.
- Waste and compost logs.
- Event transportation plans.

Synthetic data is generated by `scripts/generate_synthetic_data.py`.

## Real Data Sources Researched

- ENERGY STAR K-12 Schools: problem framing for school energy waste.
- NASEO/EIA school energy breakdown: HVAC, cooling, lighting, and ventilation assumptions.
- Building Data Genome 2: real hourly non-residential meter data for forecasting and anomaly benchmarking.
- OEDI Secondary School reference building: school-specific energy simulation assumptions.
- NREL ComStock: end-use commercial building load profiles.
- DOE Green Button: future real utility bill/interval data ingestion.
- WWF/EPA school food waste resources: food waste baselines and measurement impact.
- EPA WaterSense: water leak impact context.

Research source metadata lives in `data/raw/research_sources.json`.

## How Synthetic Data Maps To The Real Product

| Synthetic table | Future real version |
| --- | --- |
| `energy_logs.csv` | utility interval data, Green Button exports, BAS/HVAC schedule exports |
| `event_logs.csv` | school calendar, event forms, cafeteria orders, after-event reports |
| `water_logs.csv` | Aqualert edge-device alerts |
| `waste_logs.csv` | Compost AI classifications and human review corrections |
| `transport_plans.csv` | event traffic notes, attendance estimates, pickup plans |

## Training Data Plan

1. Start with synthetic data for demo and model wiring.
2. Add public BDG2 samples for energy forecasting benchmarks.
3. Add manually entered real school utility bills or Green Button data if available.
4. After each school event, log predicted vs actual attendance, food waste, energy use, and cleanup notes.
5. Retrain the event planning model once enough local event history exists.

## Responsible Data Rules

- Do not store student names or individual student routes.
- Aggregate event attendance and transportation patterns.
- Keep utility/customer data opt-in.
- Keep human approval for all physical actions.
- Let the agent cite retrieved context instead of inventing claims.
